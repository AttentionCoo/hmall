# 🚀 黑马商城 (HmMall) —— 高并发微服务电商解决方案

`HmMall` 是一个基于 **Spring Boot 3.x** 与 **Spring Cloud Alibaba** 深度构建的 B2C 分布式电商平台。项目涵盖了从用户认证、全品类商品检索、购物车联动、高并发下单、分布式事务处理、可靠性支付到延时物流追踪的全生命周期业务。

项目采用标准的微服务架构拆分，聚焦于解决**分布式数据一致性**、**高并发流量削峰**、**海量数据实时检索**等电商核心痛点，是一套生产可参考的分布式解决方案。

---

## 🛠 核心技术栈 (Technology Stack)

### 1. 微服务治理与核心架构

* **核心框架**：Spring Boot 3.x（深度利用 AOT 编译优化，提升容器启动速度与运行时效率）。
* **服务注册与配置**：Nacos（实现服务的动态发现、健康检查与配置集中式管理）。
* **流量防护**：Sentinel（提供全方位的流量控制、熔断降级与热点参数限流，保障双11级高并发场景的稳定性）。
* **统一网关**：Spring Cloud Gateway（负责动态路由转发、全局负载均衡、基于 JWT 的权限校验及跨域处理）。
* **服务调用**：OpenFeign（声明式 HTTP 客户端，配合 Ribbon 实现微服务间的高效、优雅调用）。
* **分布式事务**：Seata（专注于 AT 模式，优雅解决跨服务调用下的数据最终一致性问题）。

### 2. 数据存储与数据高可用

* **持久层核心**：MySQL 8.0 + MyBatis-Plus 3.5.x（利用 Lambda 表达式杜绝硬编码，结合行级锁与索引深度优化）。
* **高性能缓存**：Redis（承载 Token 存储、商品热点数据、秒杀库存预扣减，大幅收敛数据库压力）。
* **搜索引擎**：Elasticsearch 7.x/8.x（构建千万级商品数据的倒排索引，实现多维度高亮、分词精准检索）。

### 3. 消息队列与中间件

* **异步解耦**：RabbitMQ（实现订单超时自动关闭、库存异步释放、物流状态异构通知，解耦核心链路）。

### 4. 链路追踪与运维监控

* **可视监控**：SkyWalking / Zipkin（全链路追踪，秒级定位分布式环境下的性能瓶颈与异常节点）。
* **容器化运维**：Docker & Docker Compose（实现一键环境迁移，动态水平扩展）。

## 📦 模块拆分与架构设计 (Architecture & Modules)

项目采用**核心业务分层**与**接口解耦**的设计理念，严格规避微服务间的循环依赖：

```text
hmall
├── hm-common      // 核心公共包：工具类、全局异常拦截、通用响应对象(R)
├── hm-gateway     // 分布式网关：统一路由转发、JWT 强校验、Sentinel 限流防刷
├── hm-api         // Feign Client 接口定义：剥离解耦，彻底解决服务间循环依赖
└── hm-service     // 微服务业务集群（各模块独立数据库）
    ├── user-service     // 用户中心：用户认证、Bcrypt 密码强哈希、个人中心
    ├── item-service     // 商品中心：类目管理、商品详情、库存控制
    ├── cart-service     // 购物车：动态增删改查、实时价格校验
    ├── order-service    // 订单中心：流水下单、延时队列处理、状态机流转
    └── pay-service      // 支付中心：对接三方支付、可靠性回调处理

```

## ✨ 核心业务场景与技术落地

### 1. 分布式下单与数据最终一致性

集成 **Seata AT 模式**，在“创建订单（`order-service`）”、“扣减库存（`item-service`）”、“核销优惠券（`user-service`）”三个微服务间开启全局事务，确保在网络抖动或服务异常时数据自动回滚，保障高并发下的资产安全。

### 2. 延迟取消机制与分布式可靠通知

利用 **RabbitMQ 死信队列 (TTL + DLX)** 落地订单超时控制。用户下单后向延时队列投递消息，30 分钟未支付则自动触发死信监听：系统自动取消订单、回滚商品库存，避免恶意占单占用服务器资源。

### 3. 数据异构与高性能搜索优化

基于 **RabbitMQ 实现数据异构**。当运营后台修改商品或上下架时，异步触发消息广播，`item-service` 监听并实时更新 Elasticsearch 索引，保证 C 端用户搜索的实时性，同时实现读写分离。

## 🚀 快速启动 (Quick Start)

### 1. 克隆源码

```bash
git clone https://github.com/AttentionCasria/hmall.git
cd hmall

```

### 2. 环境初始化

1. **MySQL**：执行 `sql/` 目录下的初始化脚本，建立对应的业务数据库。
2. **Nacos**：启动 Nacos Server，并将各微服务模块 `bootstrap.yml` 中的 `server-addr` 修改为您的实际地址。
3. **Redis & RabbitMQ**：确保中间件服务正常运行并修改对应密码配置。

### 3. 编译部署

```bash
# 一键编译并安装公共依赖
mvn clean install

# 建议启动顺序：
# hm-gateway -> user-service -> item-service -> cart-service -> order-service -> pay-service

```

## 💡 个人思考与核心优化 (Optimization Highlights)

* **性能飞跃**：通过 Redis 建立**商品热点数据预热机制**，在秒杀与大促前夕将高频商品载入缓存，QPS 整体提升了约 **40%**。
* **极致解耦**：将公共 POJO、DTO 及 Feign 客户端独立剥离至 `hm-api` 模块，不仅降低了模块间的耦合度，还消除了代码冗余，提高了团队协作效率。
* **防御式编程**：针对高频访问及核心秒杀接口，配置了 **Sentinel 热点参数限流**。对刷单 IP 及恶意爬虫进行秒级熔断，有效防御了恶意刷单行为。
